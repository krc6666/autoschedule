# 自动排班产品规格

## 产品定位

自动排班是面向机场地勤保障团队的静态、本地优先排班工具，用于把航班、岗位资质、人员状态和历史疲劳合并为当天可执行的保障排班，并输出适合留档和转发的结果。

## 当前能力

- 管理人员编号、姓名、常规/行政支援类型、CX航前资质、值班资质、夜班能力、在岗状态和备注；配置页人员信息默认收起，可独立展开编辑。
- 在配置页维护可复用的航班计划模板（航班号、时段、岗位和备注）；在航班页输入航班号即可自动带出模板。
- 管理航班时段、岗位清单、岗位资质、岗位疲劳点、常规/引导/分流/行政支援分类和岗位启用旅客人数阈值；分流岗位可配置提前撤岗分钟，岗位规则按航班折叠，支持批量新增、上下调整和柜台编号一键降序。
- 导入人员、航班计划、航班配置模板、岗位配置与历史排班 Excel。
- 按航班开始时间依次排班，优先选择综合疲劳更低的可用人员。
- 自动阻止无资质、病休、夜班能力不足、时间冲突和超过每日工时上限的分配。
- 排班页按航班生成 Excel 式双列：每个航班表头横跨“岗位”和“人员”两列，岗位名称及岗位配置备注位于左格，人员姓名及当次临时备注位于右格；不同航班的岗位按各自配置从上到下独立排列，不按柜台字母或数字跨航班匹配。默认岗位列 64px、人员列 56px，航班多时横向滚动；支持在 70% 至 160% 之间缩放列宽和字号并在本机记住比例。航班标题显示到岗备注；督导置于首行，其余岗位严格按岗位配置逐条生成和排序，也不自动创建未配置岗位；柜台岗位位于分隔线上方，引导岗位统一位于下方；每区保留可直接填写岗位和任意姓名的空白新增格，临时项可修改或删除；已排人员可按住姓名跨航班拖拽交换。
- 排班页可切换行政支援模式；关闭时只显示常规体系岗位，开启后同航班同名的行政支援岗位替换常规岗位并保持为空，没有行政同名项的常规岗位继续保留。行政支援人员在独立着色名单中维护并可拖到任意岗位，不受岗位资质限制。
- 人员负荷与疲劳表只统计常规人员；在岗位数量、岗位资质和硬约束允许时，自动排班优先覆盖当天尚未获得实际工时的正常常规人员，病假和休假人员不要求产生工时。
- 排班页显示当日月度轮值：CX航前 1 人、值班 1 人、备勤 2 人，四个人选不得重合。月表只列与当前排班日期属于同一“上一休一”班组的本月工作日；CX航前仅在勾选CX航前资质的正常常规人员中轮换，值班仅在勾选值班资质的正常常规人员中轮换，备勤在全部正常常规人员中轮换，病休、休假和行政支援不参与。CX航前与值班/备勤分别展开，按人员显示本月次数和具体日期；默认值班次数最大差不超过 1，备勤按本月已排次数优先选择，月表可直接调整并实时更新统计，值班与备勤选择对方时在资质允许时自动交换，也可按日恢复默认轮值。
- 值班任务默认计 12 点疲劳，参与航班岗位候选排序、人员负荷、排班反馈和历史归档；CX航前与备勤不增加疲劳。在岗位资质和全部安全约束允许时，值班人员优先安排到最晚航班，岗位优先级依次为一号、督导、申报、送资料和其他岗位。
- 排班结果下方提供 6 条简短反馈，随自动排班和人工调整实时重算：人员覆盖、疲劳分布、航班衔接、连续高负荷、上一工作日晚班和轮值安排。反馈使用当前排班、最近工作日归档及当前策略参数，标明已满足、需复核或缺少历史基准；轮值反馈逐人说明CX航前、值班和两名备勤的实际安排，并明确值班人员是否进入最晚航班的晚撤岗位。
- 左侧第 5 个工作模块为独立的“策略”页，公开“当前排班规则清单”；每类策略和每条岗位衔接规则均默认收起，可独立展开编辑。清单展示系统实际执行的硬约束、岗位生成、分配优先级、岗位复用、岗位衔接、稳定排序、当前参数和调整入口。可配置高负荷岗位衔接保护、滚动负荷上限、同岗轮换和跨日晚班减负；另可新增多条结构化“岗位衔接间隔”规则，分别维护前序航班、前序晚撤岗位、目标航班、目标岗位、最小间隔和执行强度。“优先避开”在人手不足时允许兜底，“严格限制”会阻止违规衔接并允许岗位留空；保存后立即应用于已有排班或下一次排班。
- 将当天排班归档为历史负荷，支持按历史统计窗口继续排班。
- 历史页按工作日期分组，以只读的航班“岗位 / 人员”双列表格还原每天整体排班；最新工作日默认展开，其他日期可按需展开。
- 导出保障明细 Excel、可离线打开的 HTML 排班页和高清 PNG 排班图片。

## 核心用户路径

1. 在配置页维护人员、岗位规则和排班约束，或导入配置模板。
2. 在航班页确认当天航班、时段和岗位。
3. 选择排班日期并生成排班。
4. 在排班页查看逐条排班反馈，复核待补位、人员疲劳、航班衔接和上一工作日晚班减负，必要时人工改派。
5. 导出 Excel、HTML 或 PNG；确认当天结果后可仅归档，或点击“归档并排后天”，把当天负荷写入历史，跨过一个休息日后立即生成下个工作日排班。

## 排班规则

### 硬约束

- 只有状态为“正常”的人员可以被分配。
- 常规岗位只能分配岗位规则中列出的资质人员。
- 与配置夜班窗口重叠的航班只能分配可上夜班的人员。
- 同一人员的两个工作时段不得重叠；首尾相接不视为冲突。
- 同一人员当天累计工时不得超过配置上限，默认 12 小时。
- 常规的手动补位岗位可以保留为待补位，由排班员人工选择人员。
- 自动排班严格按岗位配置规则 ID 逐条生成常规、引导和分流岗位，不生成航班清单或其他航班中的未配置岗位，也不自动补充通用临时岗位；启用旅客人数只控制是否自动安排，低于阈值的已配置岗位仍显示为空白可编辑格。
- 行政支援模式默认关闭；关闭时不生成行政支援岗位，也不显示行政支援人员池。开启时，同航班同名的行政支援规则替换常规规则并始终留空，不能同时生成两个同名岗位；未配置行政同名规则的常规岗位仍按原规则显示和自动排班。行政支援人员不参与自动排班，可手动拖入任意岗位并绕过岗位资质限制；病休、夜班能力、时间冲突和每日工时上限仍然生效。
- 分流岗位只在原航班12:00以后开始、与下一航班的重叠不超过岗位配置的提前撤岗分钟时生效；生效后缩短原岗位结束时间并允许同一人员保障下一航班，早班不适用。
- 岗位规则的“启用旅客人数”大于 0 时，仅当当天航班预定人数达到阈值才自动安排该岗位；未达到阈值时仍生成空白岗位格，阈值为 0 表示始终自动安排。
- “引导”分类岗位严格按照该航班当前岗位显示顺序，从下向上复用已经排到常规岗位的常规人员；分流、行政支援、引导及临时岗位人员均不能成为来源。多个引导岗位依次选择不同人员，不额外占用人员，也不重复累计工时。

### 排序与疲劳

- 航班按开始时间升序处理；督导优先显示，其他岗位按岗位配置顺序处理，可对单个航班一键将 G/H 柜台按编号从大到小排列。
- 同一航班优先安排合格人员更少、限制更严格的岗位；满足策略保护后，优先选择当天尚未获得实际工时的正常常规人员，再为后续时间重叠且合格人员更少的岗位保留人选，并按历史疲劳加当天已分配岗位疲劳升序选择，分值相同时按人员编号稳定排序。
- 当日值班人员满足岗位资质、夜班能力、时间冲突、工时上限和严格保护规则后，优先保留给最晚航班的一号岗位；没有可做的一号岗位时依次选择督导、申报、送资料或其他岗位。高负荷恢复、岗位衔接、滚动负荷、同岗轮换和跨日晚班减负等已启用保护优先于这项偏好，未能进入理想岗位时由排班反馈标明结果。
- 高负荷岗位默认定义为疲劳点不少于 4，或岗位配置含“一号、申报、控制”等非空备注；默认恢复时间为 360 分钟。保护开启时，恢复时间内再次安排高负荷岗位会按配置优先避开或严格禁止，低负荷岗位不受该衔接规则影响。
- 岗位衔接间隔规则只在目标航班和目标岗位都匹配时生效；候选人此前承担指定前序航班的晚撤岗位，且从前序任务结束到目标任务开始的间隔小于规则值时，按规则优先避开或严格禁止。默认规则将 `CX931` 的督导、G20、G19 到 `TR121 H02` 的最小间隔设为 180 分钟，排班员可编辑、停用、删除或继续新增同类规则。
- 滚动负荷上限默认统计目标岗位开始前 360 分钟内已经完成的岗位疲劳；仅当即将投放的岗位属于高负荷，且窗口内疲劳加新岗位疲劳超过 8 点时触发。优先模式先选择未超限或超限更少的合格人员，严格模式排除所有投放后超限的人员。
- 同岗轮换默认回看排班日期之前 3 天的已归档记录；同一人员近期承担过同航班同岗位时，优先选择其他合格人员。优先模式在所有候选人都近期做过该岗位时允许兜底，严格模式禁止重复并可能留空；未归档的临时生成结果不进入跨日轮换判断。
- 跨工作日晚班减负默认把 20:00 以后、距离当日最晚航班开始时间不超过 180 分钟的航班视为最后一批晚班。最近一个已归档工作日的最后一批晚班承担过高负荷岗位的人员，在下个工作日最后一批晚班优先由其他人员承担靠前或疲劳点超过 2 的岗位；人手有限时，受保护人员会优先落到配置顺序靠后且疲劳点较低的岗位。优先模式在人手不足时允许兜底，严格模式禁止其承担超过下个工作日晚班疲劳上限的岗位并可能留空。
- “归档并排后天”先以当前日期替换归档记录，再把日期推进两天并调用同一排班规则；后天航班有变化时可调整航班计划后重新排班。
- 历史疲劳只统计排班日期之前、配置窗口内的记录，默认 7 天。
- 连续工作天数产生额外疲劳惩罚，默认每个连续日 5 点。
- 岗位疲劳点是当前岗位规则的最终当日计分；夜班能力是硬约束，不重复放大已经包含夜间影响的岗位疲劳点。
- 排班结果展示岗位配置备注，但隐藏“一号”字样；单元格临时备注独立保存，不覆盖岗位配置备注。

## 数据模型与生命周期

- `Staff`：人员身份、常规/行政支援类型、CX航前资质、值班资质、夜班能力、状态和备注。
- `Flight`：当天航班、时段、岗位和备注。
- `FlightTemplate`：可被当天航班按航班号套用的时段、岗位和备注预设。
- `PositionRule`：航班与岗位的资质、疲劳点、常规/引导/分流/行政支援分类、提前撤岗分钟和手动补位标记。
- `Assignment`：当天生成或人工调整后的岗位分配，分别保存岗位配置备注与当次手工备注。
- `HistoryRecord`：已确认并归档的历史工作与疲劳记录。
- `DutyRosterOverride`：按日期保存人工调整后的CX航前、值班和两名备勤；未调整日期按当月工作日顺序实时计算。
- 应用状态使用版本化 JSON 存储在浏览器 `localStorage`；损坏或不兼容的数据会回退到内置初始配置。
- 应用没有后端、账号、遥测或远端数据写入。

## Excel 合同

### 导入

识别以下工作表名称或包含这些关键词的名称：

- `人员信息`：编号、姓名、是否可上夜班、状态、备注、人员类型（常规或行政支援）、CX航前资质和值班资质。
- `航班计划`：当天航班号、开始时间、结束时间、预定人数（运力）、涉及岗位、备注。
- `航班配置` / `航班模板`：可复用航班号、开始时间、结束时间、涉及岗位、备注。
- `岗位配置`：航班号、分类、岗位名称、备注、可胜任人员、疲劳点、启用旅客人数和提前撤岗分钟。
- `各航班岗位疲劳度计算`：航班号、岗位名称、岗位疲劳点数。
- `历史排班` / `排班结果`：日期、航班、岗位、姓名、时段、工时、疲劳点和备注。

导入配置类工作表会替换对应当前配置并清空尚未归档的排班；导入历史记录按日期、航班、岗位和姓名去重合并。

### 导出

- `保障明细`：按现行业务表习惯，以航班为岗位/人员双列并排展示；人员格包含可见岗位备注与当次手工备注。
- `排班结果`：机器可读的逐岗位长表，可再次导入。
- `人员排班`：按人员展开的航班、岗位、时段和工时明细。
- 左侧“导入数据”用于混合导入配置或历史；配置页的“导入配置模板”只更新配置，历史页的“导入历史排班结果”只累计疲劳历史；左侧“导出配置”只导出配置和模板，排班页的“导出排班结果”导出当天结果。
- HTML 文件自包含样式，可离线打开、打印和转发。
- PNG 使用与 HTML 相同的航班视角和人员视角生成高清长图。

## 技术架构

- TypeScript 5.9，严格类型检查。
- Vite 7 负责开发服务和静态生产构建。
- Bootstrap 5 与 Bootstrap Icons 提供组件、布局和图标基础。
- `xlsx-js-style` 处理 Excel 导入、样式和导出。
- `html2canvas` 生成排班 PNG。
- Vitest 4 保护时间、疲劳、约束、排班、Excel 映射和状态恢复等核心逻辑。
- `src/domain/` 只处理业务规则；`src/infrastructure/` 处理浏览器存储、Excel 和分享文件；`src/ui/` 只负责界面渲染；`src/app.ts` 负责交互编排。

## 运行、验证与分发

- 开发：`npm.cmd run dev`
- 类型检查：`npm.cmd run typecheck`
- 核心测试：`npm.cmd test`
- 生产构建：`npm.cmd run build`
- 完整验证：`npm.cmd run verify`
- 构建产物：`dist/`，可部署到任意静态站点或直接作为静态包分发。
- GitHub Pages 由 `.github/workflows/deploy-pages.yml` 在 `main` 分支推送后自动构建和部署。

## 验收标准

- 完整验证命令退出码为 0。
- 默认配置可以生成排班，所有自动分配均满足硬约束。
- 人工改派无法绕过资质、状态、夜班、冲突和工时规则。
- 配置与历史 Excel 可以解析，三类分享文件可以在浏览器端生成。
- 桌面和移动端均能完成核心路径，页面无不可访问的资源或阻塞性控制台错误。
