# 自动排班产品规格

## 产品定位

自动排班是面向机场地勤保障团队的静态、本地优先排班工具，用于把航班、岗位资质、人员状态和历史疲劳合并为当天可执行的保障排班，并输出适合留档和转发的结果。

## 当前能力

- 管理人员编号、姓名、常规/行政支援类型、是否为分队长、CX航前资质、值班资质、夜班能力、在岗状态和备注；配置页人员信息默认收起，可独立展开编辑。分队长仍属于常规人员，只在名称包含“督导”的岗位自动候选中作为补缺兜底。人员状态变为病假或休假时，当前活动排班立即按最新人员状态重新计算；恢复正常后重新进入合格候选池。
- 在配置页维护可复用的航班计划模板（航班号、时段、岗位和备注）；在航班页输入航班号即可自动带出模板。
- 管理航班时段、岗位清单、岗位资质、岗位疲劳点、常规/引导/机动督导/分流/行政支援分类和岗位启用旅客人数阈值；分流岗位可配置提前撤岗分钟，岗位规则按航班折叠，支持批量新增、上下调整和柜台编号一键降序。
- 导入人员、航班计划、航班配置模板、岗位配置与历史排班 Excel。
- 12:00 以前开始的航班先汇总全部常规岗位，按每个岗位的可胜任人数从少到多全局排班；12:00 及以后开始的航班仍按开始时间处理。
- 自动阻止无资质、病休、夜班能力不足、时间冲突和超过每日工时上限的分配；同航班顶部“督导”拖入兼任范围允许的空白岗位时，是目标岗位资质、同航班时间冲突与重复工时规则的明确例外，但不得绕过机动督导兼任范围。
- 排班页按航班生成 Excel 式双列：每个航班表头横跨“岗位”和“人员”两列，岗位名称及岗位配置备注位于左格，人员姓名及当次临时备注位于右格；不同航班的岗位按各自配置从上到下独立排列，不按柜台字母或数字跨航班匹配。默认岗位列 64px、人员列 56px，航班多时横向滚动；支持在 70% 至 160% 之间缩放列宽和字号并在本机记住比例。航班标题显示到岗备注；性质为“机动督导”的岗位固定置于首行，其余岗位严格按岗位配置逐条生成和排序，只有引导岗位位于底部分隔线下方；每区保留可直接填写岗位和任意姓名的空白新增格，临时项可修改或删除；普通岗位按原约束拖拽，顶部督导只能拖入同航班兼任范围允许的空白岗位形成同步的督导兼任。
- 排班页可切换行政支援模式；关闭时只显示常规体系岗位，开启后同航班同名的行政支援岗位替换常规岗位并保持为空，没有行政同名项的常规岗位继续保留。行政支援人员在独立着色名单中维护；仅当没有满足硬约束的常规人员可用时，才允许把具备该岗位资质的行政支援人员拖入补位。
- 人员负荷与疲劳表只统计常规人员；在岗位数量、岗位资质和硬约束允许时，自动排班优先覆盖当天尚未获得实际工时的正常常规人员，病假和休假人员不要求产生工时。
- 月度轮值必须按“值班 → CX航前 → 备勤”的顺序生成。月表仍只列与当前排班日期属于同一“上一休一”班组的本月工作日。值班拥有绝对优先级并最先安排：每名有值班资质的正常常规人员在当月工作日席位充足时至少值班 1 次，在此基础上继续均衡，使值班次数最多与最少的差值不超过 1；当月值班席位少于资质人数时，无法满足的缺额跨月轮换。
- 值班与 CX航前、备勤完全互斥：同一人在同一天被安排为值班后，不得再承担 CX航前或任一备勤。CX航前与备勤不互斥，同一人在同一天可以同时承担这两项；两名备勤人员之间仍不得重复。
- 值班完成后安排 CX航前，只使用具备 CX航前资质的正常常规人员，并使所有 CX航前资质人员的当月次数最多与最少相差不超过 1。唯一 CX 资质人员值班当天没有其他人选时，值班保持不变，CX航前留空。
- 值班完成后安排备勤。每名正常常规人员在当月备勤席位充足时至少备勤 2 次，在满足保底后继续均衡，使备勤次数最多与最少的差值不超过 1；病休、休假和行政支援不参与。若当月工作日数量不足，优先满足值班刚性要求，备勤不足 2 次的人员和实际次数在月表及反馈中说明，该缺额不计入违约。
- 排班主区在人员名单和班表右侧固定显示当日CX航前、值班和两名备勤，使默认比例下可同时核对班表与轮值人选；窄屏时摘要自动移到班表上方。月度轮值明细位于班表下方，CX航前与值班/备勤分别展开，按人员显示本月次数和具体日期；月表直接显示值班、航前和备勤差值、值班首轮覆盖、备勤 2 次保底及计划疲劳。月表可直接调整并实时更新统计，值班与备勤选择对方时在资质允许时自动交换，CX航前可与任一备勤选择同一人，也可按日恢复默认轮值。
- 值班任务默认计 12 点疲劳，当日值班卡片、月度计划疲劳和排班反馈均直接显示该分值；分值同时参与航班岗位候选排序、人员负荷和历史归档，CX航前与备勤不增加疲劳。排班员可维护有序的值班岗位优先项，每项包含启用状态、航班号和岗位或备注关键词；系统按顺序锁定第一个满足岗位资质、夜班能力、时段冲突、工时上限和严格衔接限制的目标。默认顺序为 `TR121 / 一号`、`TW616 / 一号`。全部配置项不可执行时，回退为依次检查最晚、倒数第二晚航班，并在每档中按一号、督导、申报、送资料排序；目标落位前不把值班人员分给其他岗位，目标本身优先于同岗轮换、疲劳和衔接等软排序。
- 排班结果下方提供两部分反馈，随自动排班和人工调整实时重算：第一部分“航班与人员安排”报告航班数量与密集程度、人员覆盖、当日工时/疲劳差和航班衔接；第二部分“规则执行情况”逐条报告12点前岗位完整性、高负荷保护、上一工作日晚班减负、08:30 前早班值班和值班晚撤等规则，明确显示“已执行”“需复核”或“无基准”及证据。12点前反馈必须列出“已突破严格限制仍安排”、因跨航班抽调产生的源岗位空缺，或“因合格人数不足而无法填满”的具体人数和原因；值班反馈逐人说明 CX 航前、值班和两名备勤的实际安排，并提示月度轮值差异。
- 左侧第 5 个工作模块为独立的“规则”页，公开“当前排班规则清单”；每类规则和每条岗位衔接规则均默认收起，可独立展开编辑。页首可搜索规则编号、名称、分类、航班号、岗位和说明，匹配规则自动展开，清空搜索后恢复搜索前的折叠状态。清单展示系统实际执行的硬约束、岗位生成、分配优先级、岗位复用、岗位衔接、稳定排序、当前参数和调整入口。可配置值班疲劳点数、值班岗位有序优先项、机动督导兼任范围、月度轻松班次统计时段、当日/滚动工时与疲劳均衡、高负荷岗位衔接保护、滚动负荷上限、同岗轮换和跨日晚班减负；另可新增多条结构化“岗位衔接间隔”规则。“优先避开”在人手不足时允许兜底；严格限制通常阻止违规衔接，但12点前常规岗位没有未受限替代人员时允许突破并在反馈留痕；保存后立即应用于已有排班或下一次排班。
- 排班结果按自然月统计常规人员的“提前下班”和“下午无航班”。提前下班只看每人当天最后一个实际航班的截载时间，必须严格早于可配置节点才计 1 次；同航班的机动督导、引导等复用记录不得重复计数，当日值班人员排除，备勤人员照常计入。下午无航班按可配置时段判断，当天参加过实际排班且没有任何航班与该时段重叠时计 1 次，早晚有航班但下午为空仍计入；值班和备勤人员均照常计入。病假、休假、行政支援和全天无排班人员不计；当前未归档排班覆盖同日历史，页面同时显示当天明细与每人本月累计。
- 将当天排班归档为历史负荷，支持按历史统计窗口继续排班。
- 历史页按工作日期分组，以只读的航班“岗位 / 人员”双列表格还原每天整体排班；最新工作日默认展开，其他日期可按需展开。
- 导出保障明细 Excel、可离线打开的 HTML 排班页和高清 PNG 排班图片。

## 核心用户路径

1. 在配置页维护人员、岗位规则和排班约束，或导入配置模板。
2. 在航班页确认当天航班、时段和岗位。
3. 选择排班日期并生成排班。
4. 在排班页查看逐条排班反馈，复核待补位、人员疲劳、航班衔接和上一工作日晚班减负，必要时人工改派。
5. 导出 Excel、HTML 或 PNG；确认当天结果后可仅归档，或点击“归档并排后天”，把当天负荷写入历史，跨过一个休息日后立即生成下个工作日排班。

## 排班规则

### 硬约束

- 只有状态为“正常”的人员可以被分配。配置页修改人员状态后必须立即重新计算当前活动排班；加载本地数据时也必须清除旧排班快照中已经病假或休假的人员。历史归档只参与工时与疲劳计算，不得绕过当前状态把人员带入当天排班。
- 常规岗位只能分配岗位规则中列出的资质人员。
- 与配置夜班窗口重叠的航班只能分配可上夜班的人员。
- 同一人员的两个工作时段不得重叠；首尾相接不视为冲突。唯一例外是同航班顶部“督导”拖入一个空白岗位形成的督导兼任，该兼任属于同一段实际工作而非两段重叠任务。
- 同一人员当天累计工时不得超过配置上限，默认 12 小时。
- 常规的手动补位岗位可以保留为待补位，由排班员人工选择人员；但航班在12:00以前开始时，性质为“常规”的岗位即使标记手动补位也必须参与自动排班。
- 自动排班严格按岗位配置规则 ID 逐条生成常规、引导、机动督导和分流岗位，不生成航班清单或其他航班中的未配置岗位，也不自动补充通用临时岗位；启用旅客人数只控制是否自动安排，低于阈值的已配置岗位仍显示为空白可编辑格。
- 行政支援模式默认关闭；关闭时不生成行政支援岗位，也不显示行政支援人员池。开启时，同航班同名的行政支援规则替换常规规则并始终留空，不能同时生成两个同名岗位；未配置行政同名规则的常规岗位仍按原规则显示和自动排班。行政支援人员不参与自动排班；只有所有具备资质的常规人员都因病休、休假、时段冲突、夜班能力或工时上限等硬约束不可用时，才允许手动拖入具备该岗位资质的行政支援人员，且仍须通过状态、资质、夜班能力、时段冲突和每日工时上限校验。
- 分流岗位只在原航班12:00以后开始、与下一航班的重叠不超过岗位配置的提前撤岗分钟时生效；生效后缩短原岗位结束时间并允许同一人员保障下一航班，早班不适用。
- 岗位规则的“启用旅客人数”大于 0 时，仅当当天航班预定人数达到阈值才自动安排该岗位；未达到阈值时仍生成空白岗位格，阈值为 0 表示始终自动安排。12:00以前开始航班的常规岗位不受该阈值影响，始终必须尝试自动填满。
- “引导”分类岗位严格按照该航班当前岗位显示顺序，从下向上复用已经排到常规岗位的常规人员；分流、行政支援、引导及临时岗位人员均不能成为来源。多个引导岗位依次选择不同人员，不额外占用人员，也不重复累计工时。
- “机动督导”是固定在每个航班首行的独立岗位分类，并按自身资质自动安排；内置岗位和从旧状态一次性迁移的岗位全部使用“常规”，只有排班员之后明确选择时才成为机动督导，保存和重新加载不得按岗位名称擅自改变分类。规则页维护结构化兼任范围，每条规则包含启用状态、适用航班（留空表示全部航班）、匹配位置（岗位名称或岗位备注）、关键词和允许/禁止方式；禁止规则优先。某航班存在生效的允许规则时，机动督导只能兼任至少命中一条允许规则且未命中任何禁止规则的岗位；没有允许规则时，默认允许未被禁止的岗位。默认在全部航班禁止岗位备注包含“一号”“申报”“排查”的岗位。自动生成 `KE166` 排班时，如果该航班的督导被明确配置为机动督导，系统必须先在该航班仍生效且兼任范围允许的数字常规岗位中安排一名同时具备机动督导资质的正常常规人员，再把同一人同步到顶部督导；如果没有允许兼任的常规岗位，则保留该人员只担任顶部督导，并在有其他合格人员时由其他人员填充常规岗位。该预留高于值班早班、负荷与疲劳排序，但存在其他合格人员时不得主动造成其他岗位空缺。开启行政支援模式后仍执行该规则，只有实际配置了同名行政规则的岗位参与替换，常规体系的机动督导不得因其他柜台的行政替换而留空。兼任人员的数字岗位姓名保持不变，同一时段工时只计算一次，机动督导和数字岗位分别累计各自配置的疲劳点。其他航班已安排的顶部督导只能拖入同一航班兼任范围允许的空白岗位，顶部姓名保持不变，目标岗位显示与顶部同步的同一人；顶部督导改人或清空时，所有关联目标同步更新或清空。该操作不要求督导具备目标岗位资质，也不重复计算同航班工时或触发同航班时间冲突，但目标岗位仍累计自身疲劳点并在反馈列出兼任。目标已有人员、目标为另一顶部督导、跨航班或兼任范围禁止时拒绝；拖动已关联的目标岗位时同样必须通过兼任范围。自动排班后的常规岗位空缺重排不得把已关联机动督导移动到禁止岗位；加载旧排班时清除已经违反当前兼任范围的关联。重新自动排班会清除人工督导兼任。普通岗位的其他拖拽继续执行原有资质、占用与冲突规则。
- 自动选择名称包含“督导”的岗位时，人员信息中标记为分队长的正常常规人员只作为补缺兜底：所有状态、资质、夜班、时间冲突、工时上限及严格限制筛选完成后，只要仍有可用的非分队长候选，就不得自动选择分队长；没有其他可用候选时必须使用符合硬约束的分队长，不得因该优先级留下督导空位。该规则同时适用于常规督导和 `KE166` 机动督导数字岗位兼任路径，不改变分队长在其他普通柜台、CX航前、值班、备勤和人工拖拽中的既有资格与排序。排班反馈必须说明当天是否启用了分队长督导补缺。

### 排序与疲劳

- 排班结果仍按航班开始时间展示；督导优先显示，其他岗位按岗位配置顺序展示，可对单个航班一键将 G/H 柜台按编号从大到小排列。展示顺序不决定12点前岗位的计算顺序。
- 自动分配完成后，如果同一航班参与自动排班的常规岗位出现中间空缺、下方岗位已有人员，系统必须在不减少该航班已填常规岗位总数的前提下重新匹配合格人员，按岗位配置顺序优先填满上方岗位并把无法避免的空缺尽量沉到底部。重新匹配不得绕过人员状态、岗位资质、夜班能力、时间冲突、工时上限或新增严格岗位衔接违例；人工待补位、机动督导、引导和行政支援岗位不参与普通岗位匹配。`KE166` 机动督导兼任人员在常规岗位间移动时必须保留顶部督导关联和工时去重，所有人员均按移动后的岗位配置重新计算岗位备注与疲劳，引导人员随后按调整后的最下方已填常规岗位重新选取。
- 12点前常规岗位完整性是排班最高优先级：系统先提取所有开始时间早于12:00的航班中性质为“常规”的单个岗位，逐岗统计满足人员状态、岗位资质和夜班能力的可胜任人数，再按人数从少到多全局安排；同一航班内同样先排候选更少的岗位。该优先级覆盖启用旅客人数阈值和手动补位标记，12:00整开始的航班不在此范围。
- 12点前岗位先使用当前目标时段没有冲突且不超过工时上限的空闲常规人员。没有空闲候选人时，才从已经排入其他12点前航班且符合目标岗位资质、人员状态、夜班能力和工时上限的人员中抽调；抽调不复制重叠任务，原岗位改为空缺并在反馈标注“因抽调至XX航班/岗位而空缺”。系统不为此从12点以后航班抽调人员。
- 12点前的高负荷恢复、滚动负荷、同岗轮换、跨日晚班减负和岗位衔接规则均不得主动造成常规岗位空缺；“优先避开”只调整候选顺序，“严格限制”在存在未受限候选人时继续避开受限人员，在无人替代时允许突破并在反馈精确标注“已突破严格限制仍安排”。值班晚撤人员预留同样不得导致12点前常规岗位空缺。
- 状态非正常、岗位资质不符、夜班能力不符、时间冲突或超过每日工时上限仍是不可突破的硬约束。硬约束导致同时可用的合格人数少于岗位数时允许留空，反馈必须标注“因合格人数不足而无法填满”，列出缺少的岗位人数及状态、资质、夜班、冲突或工时原因；因跨航班抽调形成的源岗位空缺单独说明，不混同为疲劳保护空缺。
- 12点及以后航班按开始时间处理，并继续在岗位完整性前提下优先安排合格人员更少、限制更严格的岗位。航班宽松时优先覆盖尚未获得实际工时的正常常规人员；航班达到紧张/密集区间时，候选比较同时计算投放后的当日工时差、历史窗口滚动工时差和当日疲劳差，但这些均衡目标只能调整候选顺序，不能清空已经可以由常规人员承担的岗位。
- 当日值班人员满足岗位资质、夜班能力、时间冲突、工时上限和岗位衔接硬约束后，必须先按值班岗位有序优先项尝试落位；全部配置项不可执行时，才使用最晚航班的一号、督导、申报或送资料岗位，最晚航班没有可执行目标时再使用倒数第二晚航班的同类岗位。值班人员还必须承担一班 08:30 及以前开始的航班；两项规则均在反馈中说明命中的配置顺序、回退落位或复核原因。值班目标与岗位完整性均优先于高负荷恢复、滚动负荷、同岗轮换和跨日晚班减负。
- 高负荷岗位默认定义为疲劳点不少于 4，或岗位配置含“一号、申报、控制”等非空备注；默认恢复时间为 360 分钟。保护开启时优先安排不在恢复期的合格人员；如果恢复期限制会导致岗位空缺，则允许安排仍在恢复期的常规人员，并在反馈中标注“已超保护仍安排”。低负荷岗位不受该规则影响。
- 岗位衔接间隔规则只在目标航班和目标岗位都匹配时生效；候选人此前承担指定前序航班的晚撤岗位，且从前序任务结束到目标任务开始的间隔小于规则值时，按规则优先避开或严格禁止。12点前常规岗位无人替代时可以突破严格禁止并反馈留痕；其他时段仍按配置强度执行。默认规则将 `CX931` 的督导、G20、G19 到 `TR121 H02` 的最小间隔设为 180 分钟，排班员可编辑、停用、删除或继续新增同类规则。
- 滚动负荷上限默认统计目标岗位开始前 360 分钟内已经完成的岗位疲劳；仅当即将投放的岗位属于高负荷，且窗口内疲劳加新岗位疲劳超过 8 点时触发。系统优先选择未超限或超限更少的合格人员；如果所有候选人都会超限，则仍须选择其中负荷较低者填满岗位，不得主动留空。
- 同岗轮换默认回看排班日期之前 3 天的已归档记录；同一人员近期承担过同航班同岗位时，优先选择其他合格人员。如果所有候选人近期都承担过该岗位，则允许重复安排，不得因轮换规则留空；未归档的临时生成结果不进入跨日轮换判断。
- 跨工作日晚班减负默认把 20:00 以后、距离当日最晚航班开始时间不超过 180 分钟的航班视为最后一批晚班。最近一个已归档工作日的最后一批晚班承担过高负荷岗位的人员，在下个工作日优先减负；但减负规则不得造成岗位空缺。08:30 及以前开始的早班需要该人员填补空缺时允许安排，并在反馈中标注“已超保护仍安排”；其他时段没有替代候选人时同样以岗位完整性优先。
- “归档并排后天”先以当前日期替换归档记录，再把日期推进两天并调用同一排班规则；后天航班有变化时可调整航班计划后重新排班。
- 历史疲劳只统计排班日期之前、配置窗口内的记录，默认 7 天。
- 连续工作天数产生额外疲劳惩罚，默认每个连续日 5 点。
- 岗位疲劳点是当前岗位规则的最终当日计分；夜班能力是硬约束，不重复放大已经包含夜间影响的岗位疲劳点。
- 排班结果展示岗位配置备注，但隐藏“一号”字样；单元格临时备注独立保存，不覆盖岗位配置备注。

## 数据模型与生命周期

- `Staff`：人员身份、常规/行政支援类型、分队长标记、CX航前资质、值班资质、夜班能力、状态和备注。
- `Flight`：当天航班、时段、岗位和备注。
- `FlightTemplate`：可被当天航班按航班号套用的时段、岗位和备注预设。
- `PositionRule`：航班与岗位的资质、疲劳点、常规/引导/机动督导/分流/行政支援分类、提前撤岗分钟和手动补位标记。
- `Assignment`：当天生成或人工调整后的岗位分配，分别保存岗位配置备注、当次手工备注和仅供规则反馈使用的系统说明；系统说明不占用排班单元格的姓名或备注区域。
- `HistoryRecord`：已确认并归档的历史工作与疲劳记录。
- `DutyRosterOverride`：按日期保存人工调整后的CX航前、值班和两名备勤；未调整日期按当月工作日顺序实时计算。
- `DutyPositionPriority`：值班人员目标岗位的有序优先项，保存航班号、岗位或备注关键词及启用状态。
- `MobileSupervisorCoverageRule`：机动督导兼任岗位的结构化允许或禁止规则，保存适用航班、匹配位置、关键词和启用状态。
- 应用状态使用版本化 JSON 存储在浏览器 `localStorage`；版本 1 升级到版本 2 时一次性把所有已选“机动督导”岗位改为“常规”，版本 2 中之后手动选择的机动督导必须正常保存；损坏或不兼容的数据会回退到内置初始配置。
- 应用没有后端、账号、遥测或远端数据写入。

## Excel 合同

### 导入

识别以下工作表名称或包含这些关键词的名称：

- `人员信息`：编号、姓名、是否可上夜班、状态、备注、人员类型（常规或行政支援）、CX航前资质、值班资质和是否为分队长。
- `航班计划`：当天航班号、开始时间、结束时间、预定人数（运力）、涉及岗位、备注。
- `航班配置` / `航班模板`：可复用航班号、开始时间、结束时间、涉及岗位、备注。
- `岗位配置`：航班号、分类、岗位名称、备注、可胜任人员、疲劳点、启用旅客人数和提前撤岗分钟。
- `各航班岗位疲劳度计算`：航班号、岗位名称、岗位疲劳点数。
- `历史排班` / `排班结果`：日期、航班、岗位、姓名、时段、工时、疲劳点和备注。

导入配置类工作表会替换对应当前配置并清空尚未归档的排班；导入历史记录按日期、航班、岗位和姓名去重合并。

### 导出

- `保障明细`：按现行业务表习惯，以航班为岗位/人员双列并排展示；人员格包含可见岗位备注与当次手工备注。
- `排班结果`：机器可读的逐岗位长表，可再次导入。
- `人员排班`：按人员展开的航班、岗位、时段和工时明细。
- 左侧“导入数据”用于混合导入配置或历史；配置页的“导入配置模板”只更新配置，历史页的“导入历史排班结果”只累计疲劳历史；左侧“导出配置”只导出配置和模板，排班页的“导出排班结果”导出当天结果。
- HTML 文件自包含样式，可离线打开、打印和转发。
- PNG 使用与 HTML 相同的航班视角和人员视角生成高清长图。

## 技术架构

- TypeScript 5.9，严格类型检查。
- Vite 7 负责开发服务和静态生产构建。
- Bootstrap 5 与 Bootstrap Icons 提供组件、布局和图标基础。
- `xlsx-js-style` 处理 Excel 导入、样式和导出。
- `html2canvas` 生成排班 PNG。
- Vitest 4 保护时间、疲劳、约束、排班、Excel 映射和状态恢复等核心逻辑。
- `src/domain/` 只处理业务规则；`src/infrastructure/` 处理浏览器存储、Excel 和分享文件；`src/ui/` 只负责界面渲染；`src/app.ts` 负责交互编排。

## 运行、验证与分发

- 开发：`npm.cmd run dev`
- 类型检查：`npm.cmd run typecheck`
- 核心测试：`npm.cmd test`
- 生产构建：`npm.cmd run build`
- 完整验证：`npm.cmd run verify`
- 构建产物：`dist/`，可部署到任意静态站点或直接作为静态包分发。
- GitHub Pages 由 `.github/workflows/deploy-pages.yml` 在 `main` 分支推送后自动构建和部署。

## 验收标准

- 完整验证命令退出码为 0。
- 默认配置可以生成排班，所有自动分配均满足硬约束。
- 人工改派无法绕过资质、状态、夜班、冲突和工时规则。
- 配置与历史 Excel 可以解析，三类分享文件可以在浏览器端生成。
- 桌面和移动端均能完成核心路径，页面无不可访问的资源或阻塞性控制台错误。
