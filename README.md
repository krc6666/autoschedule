## 一张表，完成当天保障排班

自动排班把航班计划、岗位规则、人员资质、请休假状态和历史疲劳统一计算，生成可人工复核和调整的当天排班结果。所有业务数据默认留在当前浏览器，无需账号或服务器。

| 工作环节 | 当前能力 |
| --- | --- |
| 配置 | 管理常规/行政支援人员、CX航前/值班资质、航班模板、四类岗位、岗位资质、上下顺序、疲劳点和运力阈值 |
| 排班 | 按资质、状态、夜班能力、时间冲突、工时与疲劳自动分配 |
| 调整 | 空白格可直接新增任意临时岗位和姓名；已有人员可改、清空、删除或跨航班拖拽交换 |
| 接续 | 点击“归档并排后天”，保存当天负荷并跨过一个休息日生成下个工作日排班 |
| 分享 | 导出 Excel 保障明细、离线 HTML 排班页和高清 PNG 图片 |

## 数据留在本地

应用是可直接分发的静态前端工具。配置、排班和历史记录存储在浏览器本地；Excel 只在当前设备中解析和生成，不上传到远端服务。

## 配置与每日排班

先在“配置”中的“航班计划模板”填写固定的航班号、时间、岗位和备注。进入“航班”后新增当日航班，输入模板中的航班号会自动带出这些信息；每天只需填写当天预定人数（运力）并按需要调整岗位。

岗位规则中的“启用旅客人数”控制自动安排，不控制岗位是否显示。即使人数没有达到阈值，已配置的 G12 等岗位仍会显示为空白，可直接填写人员。自动排班严格按岗位配置规则逐条生成常规、引导和分流岗位；不会按名称合并同名规则，也不会从航班清单、其他航班或缺员状态中自动增加岗位。

岗位类别可选常规、引导、分流或行政支援。引导岗位按照同一航班当前显示顺序，从最下方常规岗位开始向上复用常规人员；分流、行政支援、引导和临时岗位都不能作为来源。多个引导岗位依次选择不同人员，不额外占用人员或重复累计工时。岗位性质只决定派人逻辑，不改变常规岗位的配置顺序：督导固定置顶，其他岗位按配置顺序排列；柜台在“引导岗位”分隔线上方，引导统一位于下方。每个航班可点击“柜台从大到小”，一次完成 G/H 柜台降序。分流岗位仅在下午和晚间按提前撤岗规则生效。

人员类型可选常规或行政支援。排班页的“是否启用行政支援模式”默认关闭：关闭时只使用常规体系岗位；开启后会重新排班，同航班同名的行政支援岗位替换常规岗位并保持空白，没有行政同名项的常规岗位继续保留，因此不会同时出现两个 G13 或两个督导。同一岗位需要两种模式时，应在配置中分别新增同航班、同岗位名、不同分类的两条规则。行政支援人员显示在独立的蓝色名单中，可现场新增、改名、删除并拖到任意岗位，不受岗位资质限制，但仍检查人员状态、夜班能力、时间冲突和每日工时上限。

排班结果采用与保障明细类似的双列结构：每个航班表头横跨“岗位”和“人员”两列，左侧 64px 岗位列显示岗位名称及岗位配置备注，右侧 56px 人员列显示姓名及当次临时备注。不同航班的岗位按各自配置从上到下独立排列，不会因为 G/H 编号相同而强行对齐。工具栏提供缩小、恢复 100% 和放大按钮，可在 70% 至 160% 之间同步调整列宽与字号并在本机记住比例；超出可视宽度时横向滚动查看其他航班。柜台区和引导区都保留空白新增行，可主动填写临时岗位和任意姓名，临时项可以继续修改或整项删除。人员负荷表可按当日工时、岗位疲劳、历史疲劳或总疲劳切换升序和降序。

自动排班先处理可胜任人员更少、限制更严格的岗位，并为后续时间重叠的稀缺岗位保留可胜任人员；满足这些约束后，再依据历史疲劳、当天岗位疲劳和人员编号选择人员。

在岗位数量、岗位资质和其他硬约束允许的情况下，系统会优先安排当天还没有实际工时的正常常规人员，避免有人重复承担多个航班而其他在岗人员仍为 0 工时。病假和休假人员不要求产生工时；行政支援人员不再显示在“人员负荷与疲劳”表中，但行政支援模式下的现场人员名单和拖拽能力保持不变。配置页的“人员信息”默认收起，点击标题即可展开或隐藏。

排班表下方的“排班反馈”固定显示 6 条简短结论，并随自动排班或人工改派实时更新：人员覆盖、疲劳分布、航班衔接、连续高负荷、上一工作日晚班、轮值安排。系统除检查 0 工时、待补位、疲劳和衔接外，还会逐人说明CX航前、值班和两名备勤的安排；值班人员没有进入最晚航班的一号、督导、申报或送资料岗位时会明确提示复核，月度出现某人 0 次、某人 2 次值班时也会直接列出姓名。

排班页的“当日轮值”显示 CX航前 1 人、值班人员 1 人和备勤人员 2 人，四个人选不得重合。轮值按三级优先级执行：先让所有值班资质人员完成第 1 轮，再开始第 2 轮；然后在避开当天值班人员的前提下，把CX航前资质人员的月度次数差控制在 1 以内；最后补齐备勤，尽量让每名正常常规人员达到 2 次并把次数差控制在 1 以内。

值班资质人数不超过当月工作日时，每个人至少值班 1 次。若唯一CX资质人员轮到值班，系统保留他的值班并把当天CX航前标为待配置；不会为了CX航前让其他人值第 2 次。人工调整造成某人 0 次、某人 2 次时，页面会列出具体姓名，并可点击“重新均衡本月”。工作日少于值班资质人数时，客观缺少的值班名额会跨月换人。

排班主区右侧固定显示当日CX航前、值班和两名备勤，默认比例下可与班表同时查看；窄屏时摘要会移到班表上方。月度轮值明细保留在班表下方：值班区显示本月席位、资质人数、首轮覆盖人数、备勤差值和每个人的保障状态，CX航前区显示资质人数、次数范围和航前差值。每个人的值班次数、航前次数、备勤次数、具体日期及“值班次数 × 单次疲劳”的计划疲劳均会列出；逐日月表仍可临时调整，统计会随调整更新。值班与两名备勤选择对方时会在值班资质允许时自动交换，调整行可以恢复默认轮值；人工调整造成航前或备勤差值超过 1 时，每日反馈会提示重新均衡。

CX航前只从勾选“CX航前资质”的正常常规人员中轮换；值班只从勾选“值班资质”的正常常规人员中轮换；备勤面向全部正常常规人员。病休、休假和行政支援不参与。旧本地配置和不含值班资质列的旧 Excel 会把常规人员默认视为具备值班资质，之后可在人员信息中逐人关闭。

值班任务默认计 12 点疲劳，当日值班卡片、月度计划疲劳和排班反馈都会直接显示；该分值也参与自动排班、人员负荷和归档后的历史疲劳，CX航前和备勤不增加疲劳。系统会先锁定值班人员在最晚航班的一号、督导、申报或送资料岗位，没有可执行目标时再锁定倒数第二晚航班的同类岗位；目标落位前不会让早班或普通岗位占用值班人员。这项规则优先于疲劳、同岗轮换和衔接的软排序，但不会绕过岗位资质、夜班能力、冲突、工时上限或明确的严格限制；未能执行时，轮值反馈会写明实际岗位和复核方向。下载或导出的配置 Excel 在“人员信息”工作表中同时包含“CX航前资质”和“值班资质”列。

左侧“策略”是独立的第 5 个工作模块。所有策略默认收起，点击标题可以单独展开编辑，避免规则全部铺开；“当前排班规则清单”和每一条岗位衔接规则也可以独立展开。“高负荷岗位衔接保护”默认把疲劳点不少于 4 或带有岗位备注的任务判定为高负荷，并在任务结束后的 360 分钟内保护该人员。可修改疲劳阈值、恢复分钟数、备注判定和执行强度；“优先避开”会优先换人但在人手不足时兜底，“严格限制”会宁可留空。

同一区域可以新增任意数量的结构化“岗位衔接间隔”规则。每条规则可编辑名称、启用状态、前序航班、前序晚撤岗位、目标航班、目标岗位、最小间隔和执行强度。系统预置 `CX931` 督导/G20/G19 到 `TR121 H02` 至少间隔 180 分钟的优先保护，可直接修改或删除。点击“保存并应用”后，已有排班会立即重新生成，没有排班时则从下一次自动排班开始生效。任意自然语言的新算法仍需先形成受支持的结构化规则类型，应用不会执行不可验证的自由文本。

“滚动负荷上限”默认查看目标岗位开始前 360 分钟，将窗口内已承担岗位疲劳与新岗位疲劳相加；新任务属于高负荷且累计超过 8 点时优先换人。窗口、上限和执行强度均可调整。

“同岗轮换”默认回看最近 3 天已归档排班。同一人近期做过同航班同岗位时，即使他的普通疲劳分更低，也优先从其他具备资质的人员中轮换；优先模式只有在无人可合理替换时才兜底，严格模式会宁可留空。需要先归档当天排班，次日排班才能依据真实结果执行轮换。

“跨工作日晚班减负”默认把 20:00 以后、距离当日最晚航班开始时间不超过 180 分钟的航班视为最后一批晚班。最近一个已归档工作日的这批晚班中承担过高负荷岗位的人员，在下个工作日最后一批晚班会优先避开疲劳点超过 2 的岗位；在人员有限且仍需上岗时，其他人员先承担靠前的高负荷岗位，使受保护人员尽量落到配置顺序靠后的轻岗位。晚班起点、最后一批航班范围、下个工作日晚班疲劳上限和执行强度都可以调整。

“当前排班规则清单”公开列出系统实际执行的全部规则、当前参数和调整入口，包括人员状态、岗位资质、夜班能力、时段冲突、工时上限、行政支援替换、运力阈值、稀缺岗位优先、在岗人员覆盖、月度轮值、特殊岗位衔接、高负荷保护、滚动负荷上限、同岗轮换、跨工作日晚班减负、疲劳均衡、引导复用、分流撤岗和同分排序。

当天调整完成后，点击排班页的“归档并排后天”：系统会先归档当天工时与疲劳，跨过一个休息日把日期推进两天，再以这些历史负荷自动生成下个工作日排班。后天航班有变化时，在航班页调整计划后点击“重新排班”即可。

历史页按工作日期显示完整的航班双列表格，岗位和人员排列方式与排班结果一致。最新工作日默认展开，较早日期按需展开，便于直接复核上一工作日整体安排是否合理。

左侧“导入数据”是混合入口，可识别配置和历史；配置页的“导入配置模板”只更新配置，历史页的“导入历史排班结果”只导入疲劳历史。左侧按钮明确为“导出配置”，排班页的“导出排班结果”才是当天排班结果。

## 本地开发

先安装 Node.js、Python 3 和项目依赖：

```powershell
npm.cmd install
```

运行 Python 启动器后，脚本会启动支持热更新的 Vite 开发服务，并自动用 Edge、Chrome 或 Firefox 的隐私窗口打开页面：

```powershell
python dev.py
```

需要检查生产构建结果时，使用：

```powershell
python dev.py --preview
```

可通过 `--port 5173` 指定端口。按 `Ctrl+C` 会停止当前服务。
