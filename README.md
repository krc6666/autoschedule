## 一张表，完成当天保障排班

自动排班把航班计划、岗位规则、人员资质、请休假状态和历史疲劳统一计算，生成可人工复核和调整的当天排班结果。所有业务数据默认留在当前浏览器，无需账号或服务器。

| 工作环节 | 当前能力 |
| --- | --- |
| 配置 | 管理常规/行政支援人员、CX航前/值班资质、航班模板、五类岗位、岗位资质、上下顺序、疲劳点和运力阈值 |
| 排班 | 按资质、状态、夜班能力、时间冲突、工时与疲劳自动分配 |
| 调整 | 空白格可直接新增任意临时岗位和姓名；普通岗位按原约束拖拽，督导补位可机动兼任同航班空白且具备资质的常规岗位 |
| 接续 | 点击“归档并排后天”，保存当天负荷并跨过一个休息日生成下个工作日排班 |
| 分享 | 导出 Excel 保障明细、离线 HTML 排班页和高清 PNG 图片 |

## 数据留在本地

应用是可直接分发的静态前端工具。配置、排班和历史记录存储在浏览器本地；Excel 只在当前设备中解析和生成，不上传到远端服务。

## 配置与每日排班

先在“配置”中的“航班计划模板”填写固定的航班号、时间、岗位和备注。进入“航班”后新增当日航班，输入模板中的航班号会自动带出这些信息；每天只需填写当天预定人数（运力）并按需要调整岗位。

岗位规则中的“启用旅客人数”控制12点及以后航班的自动安排，不控制岗位是否显示。12点前开始航班的常规岗位无论是否达到阈值、是否标记手动补位，都会尝试自动填满；其他岗位未达到阈值时仍显示为空白，可直接填写人员。自动排班严格按岗位配置规则逐条生成常规、引导、督导补位和分流岗位；督导补位默认显示同一航班已经排定的常规督导，不会按名称合并同名规则，也不会从航班清单、其他航班或缺员状态中自动增加岗位。

岗位类别可选常规、引导、督导补位、分流或行政支援。引导岗位按照同一航班当前显示顺序，从最下方常规岗位开始向上复用常规人员，不额外累计工时或疲劳。“督导补位”默认同步显示同一航班的常规督导，也不重复累计工时或疲劳；它可以在同一航班督导岗位间移动，也可以拖入同航班空白且已把该督导列入资质名单的普通常规岗位。拖入后常规督导姓名保留，补位槽清空，目标岗位显示该督导，表示督导机动兼任；同一航班工时只计算一次，目标岗位疲劳仍会累计并在反馈中列出。目标已有人员、无目标资质或跨航班时会拒绝。重新自动排班会清除人工兼任并恢复默认联动。督导补位仍留在原柜台配置顺序中且不显示额外分类标记，常规督导固定置顶，只有引导岗位位于底部分隔线下方。每个航班可点击“柜台从大到小”，一次完成 G/H 柜台降序。分流岗位仅在下午和晚间按提前撤岗规则生效。

人员类型可选常规或行政支援。排班页的“是否启用行政支援模式”默认关闭：关闭时只使用常规体系岗位；开启后会重新排班，同航班同名的行政支援岗位替换常规岗位并保持空白，没有行政同名项的常规岗位继续保留，因此不会同时出现两个 G13 或两个督导。行政支援人员显示在独立名单中；只有没有满足硬约束的常规人员可用时，才允许把具备该岗位资质的行政支援人员拖入补位，状态、资质、夜班能力、时间冲突和每日工时上限仍全部校验。

人员状态修改会立即影响排班。把人员从“正常”改为“病假”或“休假”后，当前活动排班会自动重新计算，该人员同时从航班岗位、CX航前、值班和备勤候选中移除；恢复“正常”后可以重新参与排班。本地保存的旧排班如果与最新人员状态不一致，重新打开应用时也会自动清除不可用人员。历史归档仍保留为过去事实，只用于工时和疲劳计算，不会把病假或休假人员复制到新排班。

排班结果采用与保障明细类似的双列结构：每个航班表头横跨“岗位”和“人员”两列，左侧 64px 岗位列显示岗位名称及岗位配置备注，右侧 56px 人员列显示姓名及当次临时备注。不同航班的岗位按各自配置从上到下独立排列，不会因为 G/H 编号相同而强行对齐。工具栏提供缩小、恢复 100% 和放大按钮，可在 70% 至 160% 之间同步调整列宽与字号并在本机记住比例；超出可视宽度时横向滚动查看其他航班。柜台区和引导区都保留空白新增行，可主动填写临时岗位和任意姓名，临时项可以继续修改或整项删除。人员负荷表可按当日工时、岗位疲劳、历史疲劳或总疲劳切换升序和降序。

自动排班先把所有12点前航班的常规岗位汇总成单个岗位队列，逐岗统计可胜任人数并从少到多安排；因此同一航班和不同航班之间都是“最缺人的单个岗位”先排，而不是先排完整个较早航班。12点及以后航班仍按开始时间处理；满足这些约束后，再依据历史疲劳、当天岗位疲劳和人员编号选择人员。

12点前常规岗位完整性是最高优先级。系统先用目标时段空闲且满足硬约束的常规人员；无人可用时，再从其他12点前航班抽调符合目标岗位硬约束的人员，不从12点以后航班抽调。抽调产生的源岗位空缺会明确反馈。疲劳和衔接严格限制在12点前无人替代时也允许突破，并标注“已突破严格限制仍安排”；只有状态、资质、夜班能力、时间冲突或工时上限等客观硬约束不足时才允许最终留空，并说明缺少人数和原因。

排班表下方的“排班反馈”分为两部分：一是“航班与人员安排”，报告航班密度、人员覆盖、工时/疲劳差和航班衔接；二是“规则执行情况”，逐条报告12点前岗位完整性、高负荷保护、上一工作日晚班减负、08:30 前早班和值班晚撤规则。12点前反馈会列出严格限制突破、跨航班抽调形成的源岗位空缺，以及合格人数不足的具体人数和原因。每条反馈都会显示“已执行”“需复核”或“无基准”，并给出当前排班证据。

排班页的“当日轮值”显示 CX航前 1 人、值班人员 1 人和备勤人员 2 人。月表严格按“值班 → CX航前 → 备勤”生成：值班最先安排，每名值班资质人员在席位充足时至少 1 次，并把次数差控制在 1 以内；随后把 CX航前资质人员的次数差控制在 1 以内；最后保证每名正常常规人员在席位充足时至少 2 次备勤，并把备勤次数差控制在 1 以内。

值班与 CX航前、备勤完全互斥，同一人当天值班后不能再承担另外两项；CX航前与备勤允许同一人当天兼任，两名备勤之间仍不能重复。若唯一CX资质人员轮到值班，系统保留他的值班并把当天CX航前标为待配置。人工调整破坏值班或轮值均衡时，页面列出具体次数并提供“重新均衡本月”。工作日少于值班资质人数时，客观缺少的值班名额跨月轮换。

排班主区右侧固定显示当日CX航前、值班和两名备勤，默认比例下可与班表同时查看；窄屏时摘要会移到班表上方。月度轮值明细保留在班表下方：值班区显示本月席位、资质人数、首轮覆盖、值班差值、备勤差值和备勤 2 次保底，CX航前区显示资质人数、次数范围和航前差值。每个人的值班次数、航前次数、备勤次数、具体日期及计划疲劳均会列出；当月备勤席位不足时，页面列出未满 2 次的人员并明确标记为“只作说明、不计违约”。

CX航前只从勾选“CX航前资质”的正常常规人员中轮换；值班只从勾选“值班资质”的正常常规人员中轮换；备勤面向全部正常常规人员。病休、休假和行政支援不参与。旧本地配置和不含值班资质列的旧 Excel 会把常规人员默认视为具备值班资质，之后可在人员信息中逐人关闭。

值班任务默认计 12 点疲劳，当日值班卡片、月度计划疲劳和排班反馈都会直接显示；该分值也参与自动排班、人员负荷和归档后的历史疲劳，CX航前和备勤不增加疲劳。系统会先锁定值班人员在最晚航班的一号、督导、申报或送资料岗位，没有可执行目标时再锁定倒数第二晚航班的同类岗位；目标落位前不会让早班或普通岗位占用值班人员。这项规则优先于疲劳和轮换等软排序，但不会绕过岗位资质、夜班能力、冲突、工时上限或明确的岗位衔接硬约束；未能执行时，轮值反馈会写明实际岗位和复核方向。

左侧“规则”是独立的第 5 个工作模块。所有规则默认收起，点击标题可以单独展开编辑；“当前排班规则清单”和每一条岗位衔接规则也可以独立展开。值班任务规则卡片中的“每次值班疲劳点”默认为 12，可直接修改并保存；当日工时与疲劳均衡卡片可配置最大差值。航班紧张或密集时，系统会比较投放后的当日/滚动工时和疲劳差，但这些目标只影响人员排序，不能为了均衡主动留空岗位。

同一区域可以新增任意数量的结构化“岗位衔接间隔”规则。每条规则可编辑名称、启用状态、前序航班、前序晚撤岗位、目标航班、目标岗位、最小间隔和执行强度。系统预置 `CX931` 督导/G20/G19 到 `TR121 H02` 至少间隔 180 分钟的优先保护，可直接修改或删除。点击“保存并应用”后，已有排班会立即重新生成，没有排班时则从下一次自动排班开始生效。任意自然语言的新算法仍需先形成受支持的结构化规则类型，应用不会执行不可验证的自由文本。

“滚动负荷上限”默认查看目标岗位开始前 360 分钟，将窗口内已承担岗位疲劳与新岗位疲劳相加；新任务属于高负荷且累计超过 8 点时优先换人，没有未超限候选人时仍选择负荷较低者填满岗位。

“同岗轮换”默认回看最近 3 天已归档排班。同一人近期做过同航班同岗位时优先选择其他具备资质的人员；如果所有候选人近期都做过该岗位，则允许重复安排，不得因轮换规则留空。

“跨工作日晚班减负”默认保护最近工作日最后一批晚班承担过高负荷岗位的人员，但不能造成岗位空缺。08:30 及以前的早班需要该人员补位时允许安排，并在反馈中标注“已超保护仍安排”；其他时段无人可替换时同样以岗位完整性优先。

“当前排班规则清单”公开列出系统实际执行的全部规则、当前参数和调整入口，包括人员状态、岗位资质、夜班能力、时段冲突、工时上限、行政支援替换、运力阈值、稀缺岗位优先、在岗人员覆盖、月度轮值、特殊岗位衔接、高负荷保护、滚动负荷上限、同岗轮换、跨工作日晚班减负、疲劳均衡、引导复用、督导补位同航班移动、分流撤岗和同分排序。

当天调整完成后，点击排班页的“归档并排后天”：系统会先归档当天工时与疲劳，跨过一个休息日把日期推进两天，再以这些历史负荷自动生成下个工作日排班。后天航班有变化时，在航班页调整计划后点击“重新排班”即可。

历史页按工作日期显示完整的航班双列表格，岗位和人员排列方式与排班结果一致。最新工作日默认展开，较早日期按需展开，便于直接复核上一工作日整体安排是否合理。

左侧“导入数据”是混合入口，可识别配置和历史；配置页的“导入配置模板”只更新配置，历史页的“导入历史排班结果”只导入疲劳历史。左侧按钮明确为“导出配置”，排班页的“导出排班结果”才是当天排班结果。

## 本地开发

先安装 Node.js、Python 3 和项目依赖：

```powershell
npm.cmd install
```

运行 Python 启动器后，脚本会启动支持热更新的 Vite 开发服务，并自动用 Edge、Chrome 或 Firefox 的隐私窗口打开页面：

```powershell
python dev.py
```

需要检查生产构建结果时，使用：

```powershell
python dev.py --preview
```

可通过 `--port 5173` 指定端口。按 `Ctrl+C` 会停止当前服务。
